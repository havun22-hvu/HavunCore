## Methoden & Frameworks

### LEES-DENK-DOE-DOCUMENTEER
Eigen werkwijze, extern geauditeerd:
1. **LEES** — Hierarchisch: CLAUDE.md → code → KB (alleen wat relevant is)
2. **DENK** — Analyseer, stel vragen bij twijfel, wacht op antwoord
3. **DOE** — Kleine atomaire wijzigingen, test voor en na
4. **DOCUMENTEER** — Nieuwe kennis op de juiste plek opslaan

### 5 Beschermingslagen
Voorkomt terugkerende fouten:
1. MD docs (waarom iets bestaat)
2. DO NOT REMOVE comments (in-code bescherming)
3. Tests (regressietests die breken)
4. CLAUDE.md regels (project-breed)
5. Memory (cross-session context)

### 5 Onschendbare Regels
1. NOOIT code schrijven zonder docs te lezen
2. NOOIT features/UI-elementen verwijderen zonder instructie
3. NOOIT credentials/keys/env aanraken
4. ALTIJD tests draaien voor en na wijzigingen
5. ALTIJD toestemming vragen bij grote wijzigingen
